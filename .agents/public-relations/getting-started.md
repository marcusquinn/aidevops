# PR Getting Started

## First move

Do not dump the full PR menu. Identify the user's immediate job and take one concrete step. If ambiguous, offer these choices:

1. Build a PR strategy.
2. Check whether a story is newsworthy.
3. Find journalists for a specific angle.
4. Monitor news for reactive opportunities.
5. Track whether a brand/keyword received coverage.
6. Review or improve a pitch.

## Routing

| User asks for | Route |
|---|---|
| “How do we get press?” | `pr-strategy.md` |
| “Is this worth pitching?” | `newsworthiness-check.md` |
| “Find reporters/journalists” | `media-list-builder.md` |
| “Does this journalist fit?” | `journalist-fit-check.md` |
| “What is happening in this beat?” | `news-search.md` |
| “Find news we can react to” | `newsjack-monitor.md` |
| “Track mentions/coverage” | `coverage-tracker.md` |
| “Write a quote/comment today” | `reactive-comment.md` |

## Mode expectations

- **Full mode**: shell/filesystem/search/browser available. Save artifacts under the repo/workspace and cite paths.
- **Limited mode**: no durable files or direct retrieval. Use pasted evidence and disclose reduced coverage.

Missing proprietary PR APIs is never a blocker. Use direct-source collection and state what could not be verified.
