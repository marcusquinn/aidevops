# Signal Agency — Do's and Don'ts

## Do

- Use one terracotta signal accent for decisions, P0, cover emphasis, and small attention marks.
- Use semantic state colours only when the colour encodes status or evidence.
- Keep tables editorial: clear headers, bottom rules, no zebra fills unless accessibility requires it.
- Put source IDs near claims and keep provenance visible in card footers or ledgers.
- Use mono metadata for dates, owners, confidence, source IDs, serials, and labels.
- Use square cards, hard rules, and paper surfaces instead of rounded SaaS panels.
- Verify contrast on all wash + solid state combinations.

## Don't

- Don't add a second brand accent, rainbow charts, decorative gradients, or Bootstrap-style state colours.
- Don't use large rounded corners, glassmorphism, soft drop shadows, or floating neumorphic panels.
- Don't make body copy state-coloured; lead with a tag or callout instead.
- Don't hide source/provenance metadata in tooltips only; reports must work in print/PDF.
- Don't stretch display type into long paragraphs or use mono for narrative copy.
- Don't overuse dossier chrome; LEDs, stamps, and tabs should reinforce trust, not become decoration.
