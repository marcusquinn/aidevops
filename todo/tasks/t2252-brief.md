# t2252: Preserve session + terminal titles across auto-compaction

## Session origin

Interactive session (maintainer-directed). User reported that after auto-compaction, the OpenCode session title, the terminal title, and the Tabby tab title all get renamed to `main`, clobbering the meaningful title set at session start.

## What

Make session + terminal titles survive OpenCode's auto-compaction cycle. Never write a default-branch name (`main` / `master` / `HEAD`) as any kind of session-scoped title, and when a regression slips through (OpenCode's internal title regeneration), restore the last-known-good title.

## Why

`t2039` added guards to `session-rename-helper.sh sync-branch` to stop it clobbering titles with `main`, but:

1. `terminal-title-helper.sh sync` (called by `pre-edit-check.sh`) and the shell-side `_aidevops_terminal_title` precmd hook (installed in `~/.zshrc` by `terminal-title-setup.sh`) both emit `repo/branch` OSC sequences with NO such guard. In the canonical repo (always on `main` per t1990), every sync fires `aidevops/main` or the bare `main` when `TERMINAL_TITLE_FORMAT=branch`.
2. OpenCode's built-in `agent=title` (haiku) regenerates session titles on events including post-compaction. When the conversation is dominated by maintenance/planning work in the canonical repo, the generator can produce `main` / `master` / `HEAD`. The t2039 guard only covers `sync-branch` — it does not defend the DB against OpenCode's own title writes.

User-observed evidence in `~/.local/share/opencode/opencode.db`: two recent sessions (`ses_25d5e9ce4ffe5JOcWIj8qrMDU2`, `ses_25d3a85ddffeMTaR9elkU4hNeN`) in the canonical repo now have title `main` despite t2039. Logs confirm both sessions went through `agent=compaction` and `session.compacted` publish events before the title regression.

## How

Four defensive layers so no single code path can regress a title:

1. **EDIT** `.agents/scripts/terminal-title-helper.sh` — add a `_is_meaningful_title` guard reused by `cmd_sync()` and `cmd_rename` (auto-title path). Never emit an OSC title that is bare `main`/`master`/`HEAD` or ends in `/main`/`/master`. Explicit `rename <title>` with a non-empty user-supplied title is unaffected (manual override, matches `session-rename-helper.sh` Test 8 semantics).

2. **EDIT** `.agents/scripts/terminal-title-setup.sh` — update the three shell integration generators (`generate_zsh_omz_integration`, `generate_zsh_plain_integration`, `generate_bash_integration`, `generate_fish_integration`) so the injected `_aidevops_terminal_title` function also skips when the branch is a default. Include a helpful comment referencing t2252 so users inspecting their rc file understand the guard.

3. **EDIT** `.agents/plugins/opencode-aidevops/index.mjs` + **NEW** `.agents/plugins/opencode-aidevops/session-title-guard.mjs` — add a `session.updated` event handler that:
   - Caches the last meaningful title per session (in-process Map, bounded).
   - When the event fires with a default-branch title AND we have a cached meaningful title AND the regression is not one we just initiated, shells out to `session-rename-helper.sh rename` to restore the cached value.
   - Dedup guard via a short-lived "restoring" set to prevent ping-pong loops.
   - Skips entirely when `AIDEVOPS_SESSION_TITLE_GUARD=0` (escape hatch).

4. **NEW** `.agents/scripts/tests/test-terminal-title-helper.sh` — mirror `test-session-rename-helper.sh`:
   - `sync` skips when branch is `main`/`master`/`HEAD` (no OSC emitted).
   - `sync` emits OSC when branch is `feature/x`.
   - `rename <title>` honours explicit titles even if title is literally `main` (user override).
   - `sync` with `TERMINAL_TITLE_FORMAT=branch` on main is also a no-op.
   - `sync` with `TERMINAL_TITLE_FORMAT=repo` still emits (just `aidevops`, which is meaningful).

## Acceptance criteria

- [ ] Plugin detects `session.updated` events where title regressed to default-branch and restores the last meaningful title within one event cycle.
